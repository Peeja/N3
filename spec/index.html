Simple triples

The simplest triple statement is a sequence of (subject, predicate, object) terms, separated by whitespace and terminated by '.' after each triple.

<http://example.org/#dt> <http://xmlns.com/foaf/0.1/name> "Dominik Tomaszuk" .

Predicate and object lists

Often the same subject will be referenced by a number of predicates.

<http://example.org/#dt> <http://xmlns.com/foaf/0.1/firstName> "Dominik" ;
                         <http://xmlns.com/foaf/0.1/lastName> "Tomaszuk" .

As with predicates often objects are repeated with the same subject and predicate.

<http://example.org/#dt> <http://xmlns.com/foaf/0.1/nick> "Domel", "Misiek"@pl .

Literals

Literals are used to identify values such as strings, numbers, dates.

@prefix : <http://example.org/vocab/> .
@prefix rdfs: <http://www.w3.org/2000/01/rdf-schema#> .
@prefix xsd: <http://www.w3.org/2001/XMLSchema#> .

_:x rdfs:comment "This is an example."^^xsd:string . # literal with XML Schema string datatype
_:x rdfs:comment "This is an example."^^<http://www.w3.org/2001/XMLSchema#string> . # same as above
_:x rdfs:comment "This is an example." .  # same again
_:x rdfs:comment "To jest przykład."@pl . # literal with a language tag
_:x rdfs:comment 'Ceci est un exemple.'@fr . # literal delimited by single quote
_:x rdfs:comment '''This # literal with embedded new lines
is
an example.''' .
                                                                            
_:y :atomicNumber 2 ; # xsd:integer
    :atomicMass 4.002602 ; # xsd:decimal
    :specificGravity 1.663E-4 ; # xsd:double
    :isNobleGas true . # xsd:boolean

We introduce several concepts which extend the syntax of RDF to N3:

The set of N3 symbols consists of the set of triple elements and the set of quantifiers.

The set of N3 triple elements consists of the disjoint sets of IRIs, blank nodes, literals, universal variables, lists, quoted graphs and of one implication sign.
